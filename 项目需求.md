# 优化后 Codex 专用小程序开发指令（精准、结构化、可直接执行）

基于 uni-app 技术栈开发，100% 适配微信小程序平台规则，覆盖原有全部业务需求，新增微信官方强制合规、授权接口、过审必备项，无歧义可直接执行。

---

## 一、基础要求（含微信上线硬性合规规则）

1. 开发**uni-app 框架微信小程序**，完全剔除原有功能（活动一览、发布活动、统计报表等全部移除）；
2. 严格遵循《微信小程序平台运营规范》《微信小程序用户信息保护规则》，所有功能符合微信官方接口权限要求，确保上线零驳回；
3. 界面适配老同志使用场景：字体默认≥16px、核心按钮点击区域≥48px*48px（微信无障碍规范）、操作极简、无冗余步骤，支持微信读屏模式无障碍适配；
4. 严格区分**普通用户（老同志）** 和**管理员**双角色权限，权限完全物理隔离，普通用户无任何管理员功能入口与访问权限；
5. 全流程符合《个人信息保护法》，敏感数据加密存储与传输，所有用户授权行为严格遵循微信「最小必要」原则。

---

## 二、页面结构与核心功能（业务需求 + 微信合规优化）

### 1. 首页（默认启动页）

- 顶部标题：自定义可配置标题
- 核心功能模块：**志愿服务**、**荣誉获奖**、（点击可进入对应子页面）
- 底部全局导航栏：固定 2 个按钮「首页」「我的」，全页面可切换

### 2. 志愿服务模块（一级入口）

下级固定 5 个子模块，不可修改：

1. 传承红色文化（关心下一代）
2. 参与基层治理
3. 服务企业发展
4. 实施以老助老
5. 其他服务

**子模块通用页面规则**：

- 页面顶部固定展示：**积分说明**（可配置文字区域）
- 下方两个核心功能按钮：**打卡**、**打卡记录**

#### （1）打卡页面（必填表单 + 微信合规校验）

所有字段带`*`为必填项，提交前强制非空校验，不可为空：

- 活动时间 *
- 活动地点 *
- 活动名称 *
- 参与内容 *
- 申报积分 *
- 上传佐证材料 *（支持图片上传，单张≤10M，最多 9 张）
- 底部：提交按钮

  申报积分根据积分规则，对不同类型的活动申报积分数值进行限制

**微信强制合规规则**：

1. 仅当用户主动点击【上传佐证材料】按钮时，才可申请微信相册 / 相机权限，禁止页面加载时提前申请权限；用户拒绝权限后，不可重复弹窗诱导，仅在用户再次触发上传时可重新申请；
2. 提交表单时，先将所有文本内容发送至后端，**强制调用微信官方 msgSecCheck 内容安全接口**完成违规文本校验；上传的图片，**强制调用微信官方 imgSecCheck 图片安全接口**完成违规内容校验；
3. 所有安全校验通过后，才可进入管理员审核列表；校验不通过的，弹窗清晰提示用户修改，禁止违规内容提交。


#### （2）打卡记录页面

- 按**年度倒序分类展示**，支持年度切换筛选
- 列表固定展示：活动名称 + 对应审核通过的有效积分
- 可点击单条记录查看详情：活动时间、地点、内容、审核状态、佐证材料

### 3. 荣誉获奖模块（一级入口）

下级固定 4 个子模块，不可修改：

1. 国家级荣誉
2. 省部级荣誉
3. 厅局级荣誉
4. 厂处级荣誉

**子模块通用页面规则**：

- 直接展示获奖信息必填表单，所有字段带`*`为必填项，提交前强制非空校验：
    
    1. 荣誉获奖名称 *
    2. 获取时间 *
    3. 授奖单位 *
    4. 荣誉积分 *
    5. 上传佐证材料 *
    
- 底部：提交按钮
- 提交规则与志愿服务打卡完全一致：先经微信内容安全接口校验，再进入管理员审核流程。
- 荣誉积分根据荣誉类型自动设置数值，无法修改

### 4. 我的页面（用户个人中心 + 微信合规入口）

- 顶部核心展示：**总积分**（大字突出展示）
- 分项积分展示：**志愿服务积分**、**荣誉获奖积分**
- 下方可视化模块：**年度积分柱状图**，按年度展示总积分趋势，支持年度切换
- 功能入口列表：
    
    1. 我的申请：查看全部待审核、已通过、已驳回的志愿服务 / 荣誉获奖记录，驳回记录可查看驳回原因并修改重提
    2. 消息订阅：自主开启 / 关闭审核结果微信通知
    3. 个人信息：查看实名认证信息，姓名 / 身份证号修改需提交管理员审核
    4. 隐私设置：查看《用户隐私保护指引》、系统权限管理、账号注销功能
    
- 微信强制合规要求：账号注销功能必须可正常使用，注销前明确告知用户后果，注销后对用户所有个人信息、业务数据进行匿名化处理或彻底删除，注销流程最长不超过 7 个工作日。

## 四、微信官方登录与实名认证体系（强制合规，上线必过）

严格遵循微信官方授权顺序，违规直接驳回，流程不可调整：

1. 底层登录逻辑完全基于**微信官方 wx.login（uni-app 对应 uni.login）接口**实现，用户唯一标识采用微信 openid（绑定微信开放平台时补充 unionid），禁止自建独立账号体系；
2. 首次启动小程序固定流程：
    
    步骤 1：启动小程序→全屏弹窗展示 **《微信小程序用户隐私保护指引》**（与微信公众平台配置审核通过的版本完全一致），明确告知用户个人信息收集范围、用途、存储期限、用户权利；
    
    步骤 2：用户必须点击【同意隐私协议】才可进入后续流程，点击【拒绝】则退出小程序，完全符合微信隐私规范；
    
    步骤 3：同意隐私协议后，自动调用 uni.login 接口获取微信临时 code，请求后端接口换取用户唯一 openid，完成微信账号绑定；
    
    步骤 4：跳转至**强制实名认证页面 **，用户需填写：姓名 *、身份证号 *（必填不可为空），提交后端完成实名核验，实名信息与微信 openid 唯一绑定，不可随意修改；
3. 非首次登录：启动小程序→校验隐私协议已同意→自动调用 uni.login 获取 openid→匹配后端实名信息→直接进入首页，无感登录；
4. 禁止首次启动强制获取用户微信昵称、头像，仅在用户主动设置个人信息时按需获取，符合微信最新头像昵称规范。

## 五、微信官方核心接口与权限集成清单

### 1. 必须集成的官方接口（上线必备）

表格

|微信官方接口|uni-app 对应 API|使用场景|微信强制授权规则|
|---|---|---|---|
|wx.login|uni.login|小程序登录、用户 openid 获取|隐私协议同意后自动调用，无需用户额外授权|
|wx.chooseImage / wx.chooseMedia|uni.chooseImage / uni.chooseMedia|佐证材料图片上传|仅用户主动点击上传按钮时申请相册 / 相机权限，禁止提前申请|
|wx.msgSecCheck|后端服务端调用|校验用户提交的文本内容，拦截违规信息|所有用户提交的文本必须校验通过后才可进入审核流程，禁止前端调用|
|wx.imgSecCheck|后端服务端调用|校验用户上传的图片，拦截违规内容|所有用户上传的图片必须校验通过后才可存储展示，禁止前端调用|
|wx.requestSubscribeMessage|uni.requestSubscribeMessage|审核结果通知订阅|用户提交申请后弹窗引导订阅，仅用户主动触发时调用，禁止强制订阅|
|wx.sendSubscribeMessage|后端服务端调用|审核通过 / 驳回后推送微信服务通知|仅用户已订阅的前提下推送，禁止无订阅推送|

### 2. 权限申请铁律（微信驳回重灾区）

1. 所有权限申请必须遵循「**先触发场景，后申请权限**」原则，禁止页面一加载就申请权限；
2. 用户拒绝权限后，不可立即重复弹窗申请，需引导用户在需要使用功能时，主动点击按钮再次申请；
3. 申请权限时，必须明确告知用户权限使用用途，不得隐瞒。

## 六、管理员专属功能（核心权限 + 微信合规优化）

### 1. 管理员身份鉴权（微信合规隔离）

- 管理员身份与微信 openid 唯一绑定，仅后端提前配置在白名单内的微信 openid，才可看到并访问管理员功能入口，普通用户完全无感知、无入口；
- 管理员登录后，需完成二次身份确认，确保账号安全；
- 所有管理员敏感操作（导入数据、修改积分、调整荣誉级别、导出数据、审核操作）全程留痕，生成不可篡改的操作日志，包含：操作人 openid、操作时间、操作内容、操作 IP，符合数据安全可追溯要求。

### 2. 批量数据导入功能

- 提供**标准化 Excel 模板下载**，模板字段严格匹配「志愿服务 5 个子模块」「荣誉获奖 4 个子模块」，包含：用户姓名、身份证号、对应模块、活动 / 荣誉名称、时间、地点、内容、积分、佐证材料链接等必填字段；
- 支持管理员批量上传 Excel 文件，后端自动解析，通过姓名 + 身份证号匹配对应用户微信 openid，匹配成功后自动录入系统；
- 导入数据默认标记为「管理员统一录入」，无需用户自行填写，自动同步展示在对应用户的志愿服务 / 荣誉获奖页面、打卡记录、个人积分中；
- 导入数据默认审核通过，直接生效计入积分，同时生成导入操作日志。

### 3. 数据审核功能

- 审核列表分类展示：待审核志愿服务申请、待审核荣誉获奖申请、已审核申请、已驳回申请；
- 支持单条 / 批量审核操作：**通过、驳回（需填写驳回原因）、修改申报积分、调整荣誉获奖级别**；
- 审核操作完成后，后端自动调用微信订阅消息接口，给对应用户推送审核结果通知（用户已订阅的前提下），同步更新用户积分与数据；
- 驳回的申请，用户可在「我的 - 我的申请」中查看驳回原因，修改后重新提交审核。

### 4. 全量数据导出功能

- 支持一键导出**全体用户 / 指定筛选用户**的完整数据，包含：用户基本信息、志愿服务全量记录、荣誉获奖全量记录、各项积分明细、审核状态、操作记录等；
- 导出格式为 Excel 文件，支持按年度、模块、审核状态、用户信息等多维度筛选导出；
- 导出操作全程留痕，生成操作日志，符合数据安全规范。

## 七、数据与交互规则

1. 所有用户数据与微信 openid 唯一绑定，用户间数据完全隔离，仅本人与管理员可查看；
2. 用户敏感信息（姓名、身份证号）采用 AES 加密存储与传输，禁止明文存储，符合微信数据安全规范；
3. 积分自动统计规则：审核通过的志愿服务积分 + 审核通过的荣誉获奖积分 = 个人总积分，实时同步更新；
4. 所有数据按年度自动归类，年度积分柱状图实时更新；
5. 禁止任何诱导分享、诱导关注、诱导下载的行为，完全符合微信小程序运营规范；
6. 所有页面加载、提交、成功 / 失败操作，均有清晰的反馈提示，无空白加载、无响应操作。
